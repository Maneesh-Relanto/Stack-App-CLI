# Generated Samples Directory

Real-world examples of all 15 boilerplate templates generated by Stack-App-CLI.

## Purpose

This directory contains sample applications generated from each template. These serve as:
- **References** - See exactly what each template produces
- **Learning Examples** - Understand project structure and best practices  
- **Validation** - Confirms generators work correctly
- **Documentation** - Real code backing the TEMPLATES_GUIDE.md

## What's Tracked

✅ **Source Code** - All `.js`, `.ts`, `.py`, `.go`, `.rs`, `.cs`, `.ex` files  
✅ **Config** - `package.json`, `go.mod`, `Cargo.toml`, `.csproj`, `mix.exs`  
✅ **Documentation** - README.md, Dockerfile, docker-compose.yml  
✅ **GitHub Actions** - CI/CD workflows  

❌ **NOT Tracked** - Dependencies, lock files, build artifacts

## What To Ignore

See `.gitignore-templates` for comprehensive rules, but basically:

```
- node_modules/
- package-lock.json, yarn.lock, pnpm-lock.yaml
- venv/, vendor/, target/, bin/, obj/
- .env  
- *.log, build artifacts
```

## How to Use

### Explore a Template

```bash
cd generated-samples/typescript/nextjs-saas-sample/
cat README.md              # Start here
cat package.json           # See dependencies
ls -la src/                # Browse structure
```

### Install & Run (example: FastAPI)

```bash
cd generated-samples/python/fastapi-modern-sample/

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Run
python main.py
```

### Use as Starter Project

```bash
# Copy the sample you like
cp -r generated-samples/typescript/nextjs-saas-sample ./my-project
cd my-project

# Fresh install
rm -rf node_modules package-lock.json
npm install

# Start developing
npm run dev
```

## For Developers Using CLI

When you run `npx create-stack-app`, your generated project will use:
- The **same structure** as samples in this folder
- The **same dependencies** as shown in package.json/requirements.txt  
- The **same configuration** as Dockerfile and .env.example

Refer to these samples to understand your project better!

## Maintenance

These samples are regenerated whenever:

- Adding new templates (Tests #16+)
- Fixing generator bugs
- Improving boilerplate quality

Simply commit the updated samples to keep them current.
